# 📘 JobTrack Pro：全生命周期求职策略管理中心

> **定位：** 专为高压求职场景设计的“降熵”工具，通过数据驱动与容错设计，将混杂的求职碎片转化为清晰的决策路径。

***

## 一、 产品定义与深度洞察 (Product Vision & Insights)

### 1. 核心痛点拆解

在传统的求职管理中，用户面临的不仅是“进度记录”问题，更是以下深层痛点：

- **认知负荷溢出：** 面对 50+ 投递目标，大脑无法实时排序优先级，导致死线焦虑（Deadline Anxiety）。
- **资产回溯断层：** “我当时投的是哪个版本的简历？”——简历版本与岗位需求的匹配关系随时间模糊。
- **决策盲目性：** 缺乏量化反馈，无法直观感知“简历转化率”与“面试转化率”的瓶颈在哪。
- **容错率极低：** 在焦虑状态下，一次误删可能意味着丢失一个核心机会。

### 2. 用户价值主张

**JobTrack Pro** 不仅仅是一个 Kanban，它是一个具备感知（智能预警）、决策（数据漏斗）与保障（安全回收）能力的求职辅助系统。

***

## 二、 核心功能模块：业务闭环设计 (Core Functionality)

### 1. 视觉化进度管理 (Lifecycle Kanban)

- **标准化状态流转：** 采用经典的 `愿望清单 -> 已投递 -> 测评中 -> 面试中 -> 已录用` 五阶段模型，将线性的时间线转化为空间状态分布，实现全局掌控。
- **负向路径闭环：** 增加 `被拒绝` 独立泳道，支持“失败归因”标记，为后续复盘提供原始数据。

### 2. 智能焦虑干预系统 (Smart Alert Engine)

- **三级动态预警：**
  - 🔴 **Critical (24h)：** 红色预警，强力唤起注意，防范死线遗漏。
  - 🟠 **Warning (3d)：** 橙色缓冲，提示用户进入备战状态。
  - 🟢 **Safe (>3d)：** 保持静默，减少非必要干扰。
- **面试战时指挥 (War Room Mode)：** 在 `面试中` 卡片正面外露**实时倒计时**与**一键会议入口**，极致压缩面试前 5 分钟的准备动作。

### 3. 简历资产化配置 (Resume Asset Management)

- **版本控制系统：** 用户可自定义管理简历版本（如：产品岗-大厂专版、算法-通用版）。
- **多维关联：** 实现“简历版本-投递渠道-岗位信息”的 1:1:1 深度绑定，解决面试前的信息回溯难题。

### 4. 数据驱动复盘 (Decision Support)

- **求职转化漏斗 (Conversion Funnel)：** 实时可视化的数据统计，量化投递表现，辅助用户决定是该“优化简历”还是“加强面试技巧”。
- **高优 Top 3：** 算法自动提取全局最紧迫任务，在海量数据中实现信息降噪。

### 5. 极端容错机制 (Error-Tolerance)

- **二级缓冲回收站 (Recycle Bin)：** 针对用户误操作逻辑，设计“逻辑删除”机制。数据不消失，只从看板隐藏，提供“后悔药”式的数据资产保护。
- **数据主权：** 支持 JSON 全量导入/导出，确保求职记录的持久性与跨设备流动性。

## 三、 交互哲学与设计美学 (UX & Design Philosophy)

### 1. 极致信噪比与渐进式披露 (Progressive Disclosure)

在大规模数据管理场景下，信息过载是用户流失的主因。本项目遵循“不需即隐藏”的原则：

- **沉浸式看板：** 默认状态下，卡片仅保留公司、岗位、关键状态和时间等“信号”信息，隐藏备注预览、复杂操作按钮。
- **悬停反馈 (Hover Effects)：** 只有当用户鼠标悬停于卡片时，相关的“写复盘”、“查面经”及详细备注才会通过淡入动画浮现。这大幅提升了用户在进行“全局扫视”时的效率。

### 2. 品牌共情设计 (Branding & Empathy)

- **Meituan Design Language：** 全站采用美团绿（#00C3AA）作为核心行动色，配合 12px 的大圆角规范与微妙的阴影（Soft Shadow），营造专业、稳重且具备亲和力的职场工具氛围。
- **状态色语义化：** 每一个看板列头部均设有对应的状态色条，利用色彩心理学降低用户对状态流转的认知成本。

### 3. 操作路径唯一化 (Consistency)

- **动静分离：** 状态切换（高频）位于卡片右下角，一键直达；详情修改（低频）收纳在“更多操作”菜单中。这种设计模仿了工业级 SaaS（如 Linear, Jira）的操作习惯，确保用户无需学习即可上手。

## 四、 工程交付与技术亮点 (Engineering Excellence)

### 1. AI 驱动的敏捷研发 (AI-Augmented Development)

本项目通过 **Trae (AI 编程助手)** 协同完成。在这种全新的研发模式下：

- **人类定义逻辑：** 我负责全局架构设计、状态机逻辑建模以及极端用户场景的预判。
- **AI 负责落地：** 利用 AI 的执行力，在极短时间内完成了 100% 的 CRUD 逻辑、复杂的 Date 处理函数以及高度一致的 Tailwind UI 样式。
- **成果：** 交付周期缩短了 70% 以上，且保证了代码的模块化与整洁度。

### 2. 健壮的持久化策略

- **LocalStorage + 数据主权：** 弃用复杂的后端依赖，采用轻量级客户端存储。通过 JSON 导出/导入功能，用户可随时备份或迁移自己的求职资产，真正实现了“离线可用，数据归心”。
- **容错与原子更新：** 每次操作（状态更新、删除、编辑）均触发原子化的 State 变更与持久化同步，确保数据在页面刷新后依然保持绝对一致。

## 五、 未来演进规划 (Roadmap - V2.0)

作为产品经理，我为 JobTrack Pro 预设了更具想象力的迭代路径：

1. **AI 简历匹配雷达：** 利用大模型分析卡片内的 JD 内容，自动计算当前简历版本的“匹配分”并给出优化建议。
2. **自动化录入插件：** 开发 Chrome 浏览器插件，实现在 Boss直聘、美团官网等平台的一键抓取，彻底消除手动录入的成本。
3. **智能面试日程表：** 接入系统日历 API，将“面试倒计时”与手机日历实时联动，实现多端同步的震动提醒。
4. **失败归因大数据：** 聚合所有用户的 `REJECTED` 原因数据，为后续用户提供该岗位的“排雷指南”。

## 六、 测评人快速测试手册 (Evaluation Guide)

为了让您能够快速感知产品的深度，建议进行以下操作：

1. **体验预警：** 观察 Mock 数据中日期临近的卡片，感受红/橙警示的视觉压迫感。
2. **体验资产管理：** 点击“简历版本”旁的设置图标，添加一个新的版本，观察下拉菜单的实时联动。
3. **体验极端容错：** 尝试删除一张卡片，然后打开顶部导航栏的“回收站”进行一键恢复。
4. **体验检索效率：** 在搜索框输入关键词，观察 5 个泳道的卡片如何同步、瞬时地完成过滤。
